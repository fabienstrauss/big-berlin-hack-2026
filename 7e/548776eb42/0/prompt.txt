Why you see this message
Your Google Cloud project is inheriting a security policy from a parent folder or organization that explicitly disables the creation of API keys. This is a security best practice designed to prevent the use of long-lived, easily exfiltrated credentials in favor of more secure methods like Application Default Credentials (ADC).

How to remove the restriction
To enable API key creation, you must have the Organization Policy Administrator role (roles/orgpolicy.policyAdmin) at the level where the policy is applied (likely the folder or organization level, as no specific policies were found directly on your project bigberlin-hack26ber-3202).

Identify the Constraint: The most common constraint causing this is constraints/serviceuser.apiKeysConfig.
Modify the Policy:
Navigate to the Organization Policies page in the Google Cloud Console.
Search for "Restrict API Key creation" or serviceuser.apiKeysConfig.
Select Manage Policy.
Choose to Override the parent policy and set the enforcement to Off, or add your specific project to an allowed exceptions list if using a custom constraint.
Wait for Propagation: Policy changes can take a few minutes to propagate before the "API Keys are Disallowed" message disappears from your current view.
Recommended Path: Use ADC Instead
Google Cloud strongly recommends using Application Default Credentials (ADC) instead of API keys. ADC automatically finds and uses credentials based on the environment (e.g., your local machine, a GCE instance, or a Cloud Run service), which is significantly more secure than managing secret strings manually.

To set up ADC on your local machine, you can run:
