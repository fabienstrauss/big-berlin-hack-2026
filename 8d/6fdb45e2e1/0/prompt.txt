## Mistake 2 — Polling Loop JSON Error

Your Cloud Shell terminal is showing repeated `JSONDecodeError: Expecting value: line 1 column 1` — this means the curl response is **empty** because `gcloud auth print-access-token` returns an expired or invalid token mid-loop. The Python JSON parser receives an empty string and crashes. [docs.cloud.google](https://docs.cloud.google.com/sdk/auth_success?cloudshell=true)

**Fixed polling script:**
```bash
OPERATION_ID="d57b19b6-dc33-4b6e-936f-9405d2a35915"
PROJECT="bigberlin-hack26ber-3202"

echo "⏳ Polling Veo operation..."
while true; do
  TOKEN=$(gcloud auth print-access-token 2>/dev/null)
  
  if [ -z "$TOKEN" ]; then
    echo "❌ Token expired, re-authenticating..."
    gcloud auth login --brief
    continue
  fi

  RESULT=$(curl -s \
    "https://us-central1-aiplatform.googleapis.com/v1/projects/${PROJECT}/locations/us-central1/publishers/google/models/veo-3.0-fast-generate-001/operations/${OPERATION_ID}" \
    -H "Authorization: Bearer ${TOKEN}")
  
  # Guard against empty response
  if [ -z "$RESULT" ]; then
    echo "⚠️ Empty response, retrying in 15s..."
    sleep 15
    continue
  fi

  DONE=$(echo "$RESULT" | python3 -c "
import sys, json
try:
    d = json.load(sys.stdin)
    print(d.get('done', False))
except:
    print('ERROR')
" 2>/dev/null)

  if [ "$DONE" = "True" ]; then
    echo "✅ Done!"
    echo "$RESULT" | python3 -m json.tool
    break
  elif [ "$DONE" = "ERROR" ]; then
    echo "⚠️ Bad response: $RESULT"
    sleep 15
  else
    echo "⏳ Still generating... ($(date +%H:%M:%S))"
    sleep 15
  fi
done
```
